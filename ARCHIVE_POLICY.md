# ARCHIVE-ONLY Repository Policy

Version: 1.0
Status: Approved
Repository Type: Historical Archive

---

# Purpose

The ARCHIVE-ONLY repository serves as the permanent historical storage location for all archived materials from active repositories.

Its purpose is to preserve project history while keeping active development repositories clean, efficient, and focused on current production work.

This repository is **not** a software development repository.

---

# Primary Objectives

The ARCHIVE-ONLY repository exists to:

- Preserve historical production releases
- Preserve historical scripts
- Preserve historical reports
- Preserve previous specifications
- Preserve documentation
- Preserve exported PDFs
- Preserve historical dashboards
- Preserve audit records
- Preserve recovery artifacts

---

# Source of Truth

Nothing contained within this repository shall be considered the current production source unless explicitly identified.

Current production always resides in the active project repository.

Examples:

- ML
- ML-Archive
-

The active repository is always the governing authority.
