# ARCHIVE-ONLY

## Purpose

ARCHIVE-ONLY is the centralized historical repository for all projects maintained by this account.

This repository stores:

- Historical production scripts
- Archived releases
- Previous framework versions
- Dashboard reports
- Timing reports
- Validation reports
- Specifications
- PDFs
- Documentation
- Legacy reference material
- Deprecated code
- Working backups

This repository is **NOT** an active development repository.

---

# Repository Policy

The files contained in this repository are maintained for:

- Historical reference
- Version recovery
- Change tracking
- Audit purposes
- Documentation

Files in this repository **must never be considered the current production source** unless specifically identified as such.

---

# Active Development

Active development occurs in the individual project repositories.

Examples:

- ML
- HOPE
- CPAP
- Billing
- etc.

Only the active repositories contain the current production code.

# Archive Rules

Files may include:

- superseded production scripts
- previous releases
- historical reports
- archived PDFs
- previous specifications
- exported documentation
- legacy code
- backup files

Files should **not** be modified except to:

- organize
- document
- preserve history

---

# AI / Agent Guidance

This repository is intended for archival purposes only.

AI assistants should:

- Treat all contents as historical reference.
- Do not assume any file is current production.
- Do not compare archive files against active repositories unless explicitly instructed.
- Do not generate updates based on archived files unless requested.
- Do not use archived scripts as the default source for new development.

When assisting with software development, always use the active project repository instead.

---

# Source of Truth

The source of truth for any project is its active repository.

ARCHIVE-ONLY exists solely to preserve historical artifacts and enable recovery when needed.

---

# Maintenance

Recommended maintenance:

- Organize by repository.
- Organize by release/version.
- Preserve original filenames whenever possible.
- Do not delete historical releases without verification.
- Add a short README to each project folder describing its contents.
